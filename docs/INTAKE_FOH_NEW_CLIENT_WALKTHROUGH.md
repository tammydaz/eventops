# Intake/FOH — New Client Call Walkthrough

**Start here:** Intake/FOH landing page (`/intake-foh`).

---

## What you see on the landing page

- **Sidebar:** Dashboard · **Intake / FOH** (section) · Add Event · Event Overview · Open BEO Intake · Leads  
- **Main area:** **Intake/FOH — 10-Day Pipeline** (event cards for the next 10 days).  
  - Subtitle: *"Click an event to open Event Overview (booking steps)"*  
- **Toolbar:** View (Grid / List / Calendar), event count, guest count.  
- **Today’s Tasks** (collapsible): tasks due today; click an event name to open Event Overview for that event.

---

## New client calls — step-by-step

### 1. Start on Intake/FOH

- Go to **Intake / FOH** in the sidebar (or open `/intake-foh`).
- You land on the **10-Day Pipeline** (or Event Overview, depending on nav).

### 2. Create the event (first contact)

- Click **Add Event** in the sidebar.
- You go to **Quick Intake** (name, phone, type, date).
- Enter:
  - **First name** · **Last name**
  - **Phone**
  - **Event type** (Full Service, Delivery, Pickup, etc.)
  - **Event date** (optional)
- Click **Create Event →**.
- The system creates the event and sends you to **BEO Intake** for that event.

### 3. You’re now on BEO Intake for the new event

- **Header:** Event selector (shows the new client/event), **Event Overview** link, Werx logo.
- You can:
  - **Fill more now:** client/contact, venue, menu, etc. (side-by-side or “Fill on BEO”).
  - **Or** click **Event Overview** to switch to the booking command center.

### 4. Event Overview (booking steps)

- Click **Event Overview** in the header (or go to Intake/FOH → **Event Overview** → click this event in the pipeline).
- You see:
  - **Event Summary** — client, event name, date, type, guests, FOH status.
  - **Booking Steps** — checkboxes driven by data:
    - Send invoice (Mark done when sent)
    - Secure deposit / payment
    - Contract sent · Contract signed
    - Send client questionnaire
    - Confirm contact details
  - **Actions** — Send Questionnaire, Add Reminder, Log Call Note, **Open Full Intake**.
  - **Documents** — Upload (invoice, contract, etc.).
  - **Outstanding Tasks** — follow-ups and reminders.
  - **Event Notes** — BEO notes (timeline, call notes, etc.).

Use this page to track the booking and do the next action (send invoice, set reminder, upload doc, open full intake).

### 5. Fill the rest of the BEO over time

- From Event Overview, click **Open Full Intake** to go back to **BEO Intake**.
- Fill client/contact, venue, menu, beverages, serviceware, timeline, notes as you get info (from the client, from emails, from site visits).
- Use **Event Overview** anytime to mark steps done (invoice sent, deposit, contract, questionnaire) and add reminders so nothing falls through.

### 6. When the BEO is complete

- In **BEO Intake**, finish all required sections.
- Use **Send to BOH** (or “Fire Event” when that’s live) so the event is finalized and automations run for Kitchen, Flair, Delivery, Ops.
- Departments then see the event and the complete BEO in their views.

## Quick reference

| I want to…              | Do this…                                                                 |
|-------------------------|---------------------------------------------------------------------------|
| Add a new event (call)  | Intake/FOH → **Add Event** → Quick Intake → fill → Create Event →        |
| Track an existing event | Intake/FOH → **Event Overview** → click event in pipeline                |
| Fill / edit BEO         | **Open BEO Intake** → pick event, or from Event Overview → **Open Full Intake** |
| See booking steps       | Open **Event Overview** for that event                                   |
| Upload invoice / doc    | Event Overview → **Documents** → Upload Document                         |
| Set a follow-up         | Event Overview → **Actions** → Add Reminder, or Tasks → Follow Up       |

---

## Where “Add Event” goes

- **Intake/FOH** → **Add Event** in the sidebar → **Quick Intake page** (`/quick-intake`).  
- After you create the event, you are redirected to **BEO Intake** for that event.  
- From BEO Intake you can go to **Event Overview** for the same event via the header link.
